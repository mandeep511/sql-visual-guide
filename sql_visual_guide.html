SQL — A Visual Guide

Everything here is explained step by step. No shortcuts, no assumptions. Read top to bottom.

The most important thing to know before you start

In every example below, you'll see names like lead_name, deal_value, leads, etc.

These are NOT fixed words. They are names that someone made up when they created the database. Your database will have completely different names — like customer_name, order_total, or products.

The only words that are fixed are the blue words — those are SQL commands (SELECT, FROM, WHERE, etc.). Everything else changes depending on your data.

Part 1
SELECT + FROM
Every SQL query starts with these two. Always. No exceptions.
The pattern (this is the shape — memorise this)
SELECT column_names_you_want FROM table_name;
What each word does
SELECT Tells SQL which columns to show you. Columns = the vertical headings in your data (like columns in Excel).
FROM Tells SQL which table to look in. A table = one sheet of data (like one tab in Excel).
* Star means "all columns". So SELECT * means "show me everything".
; Semicolon = full stop. It tells SQL "I'm done writing my query".
You have an Excel sheet. You want to see only some columns. You'd hide the rest. SELECT does exactly that — it picks which columns to display.
Example with sample data
SELECT lead_name, region, deal_value FROM leads;
What SQL returns
lead_nameregiondeal_value
Riya MehtaWest800,000
Arjun NairNorth1,500,000
Priya ShahSouth650,000
Karan BoseWest1,200,000
Same idea, different names
SELECT product_name, price FROM products;
See? Different table, different columns. The pattern is the same. Only the green names changed.
Part 2
WHERE
Filters your data. Only keeps rows that match a condition you set.
The pattern
SELECT columns FROM table WHERE column operator value;
What each word does
WHERE Starts the filter. Only rows that pass this test will show up.
AND Add more conditions. ALL of them must be true.
OR ANY one condition being true is enough.
= > < >= <= != Comparison operators. = means "equals", != means "not equal to", > means "greater than".
'text' Text values must be wrapped in single quotes. Numbers don't need quotes.
Excel's Filter button. Click a column, pick a value, and only matching rows stay visible. WHERE does the same thing — but you type the filter instead of clicking it.
Example
SELECT lead_name, deal_value FROM leads WHERE region = 'West' AND deal_value > 500000;
How SQL processes each row
lead_nameregiondeal_valueResult
Riya MehtaWest800,000kept
Arjun NairNorth1,500,000dropped — North ≠ West
Priya ShahSouth650,000dropped — South ≠ West
Karan BoseWest1,200,000kept
Same idea, different names
SELECT student_name, marks FROM exam_results WHERE marks >= 90;
Completely different table and columns. The pattern is identical — SELECT → FROM → WHERE.
Part 3
Aggregate Functions
These are built-in calculators. They take many rows and return one number — a count, a total, an average, etc.
Aggregate functions are SQL keywords (blue words). They are always spelled exactly the same: COUNT, SUM, AVG, MIN, MAX.
But the column name you put inside the brackets — that depends on YOUR data.
The pattern
SELECT FUNCTION_NAME(column_name) FROM table_name;
You replace column_name with whatever column you want to calculate on.
Let's look at each one using this sample data:
leads table (4 rows)
lead_nameregiondeal_value
Riya MehtaWest800,000
Arjun NairNorth1,500,000
Priya ShahSouth650,000
Karan BoseWest1,200,000
COUNT( )
Counts how many rows there are.
SELECT COUNT(*) FROM leads;
Returns → 4
COUNT(*) counts all rows. COUNT(column) skips empty cells.
SUM( )
Adds up all the values in a column.
SELECT SUM(deal_value) FROM leads;
Returns → 4,150,000
800k + 1500k + 650k + 1200k = 4,150,000
AVG( )
Calculates the average (mean).
SELECT AVG(deal_value) FROM leads;
Returns → 1,037,500
4,150,000 ÷ 4 rows = 1,037,500
MIN( )
Finds the smallest value.
SELECT MIN(deal_value) FROM leads;
Returns → 650,000
The smallest deal_value in the table.
MAX( )
Finds the largest value.
SELECT MAX(deal_value) FROM leads;
Returns → 1,500,000
Same functions, different data
SELECT MAX(salary) FROM employees;
Same MAX. Different column, different table.
Naming your result with AS

When you use an aggregate function, SQL doesn't know what to call the result column. AS lets you give it a name — any name you want.

SELECT COUNT(*) AS total_leads, SUM(deal_value) AS total_revenue, AVG(deal_value) AS average_deal FROM leads;
total_leadstotal_revenueaverage_deal
44,150,0001,037,500
The names after AS (like total_leads) are names you make up. You could write AS banana and it would work — it would just be a confusing name. Pick names that describe what the number means.
Part 4
GROUP BY
Takes your rows, puts them in groups based on a column, then runs aggregate functions on each group separately.
The pattern
SELECT grouping_column, FUNCTION(column) AS name FROM table GROUP BY grouping_column;
What each word does
GROUP BY Tells SQL: "put all rows with the same value in this column into one group". Then each function runs once per group.
If you use GROUP BY, every column in SELECT must either be (a) the column you're grouping by, or (b) inside an aggregate function. You can't just list a random column — SQL wouldn't know which row's value to show.
A Pivot Table in Excel. You drag "Region" to the Rows area, and "Deal Value" to the Values area with "Sum". GROUP BY is the same — group by region, sum the deal values.
Example
SELECT region, COUNT(*) AS total_leads, SUM(deal_value) AS revenue, AVG(deal_value) AS avg_deal FROM leads GROUP BY region;
Step 1 — SQL groups the rows by region
West Riya (800k) + Karan (1,200k)
North Arjun (1,500k)
South Priya (650k)
↓
Step 2 — Each group collapses into one summary row
regiontotal_leadsrevenueavg_deal
West22,000,0001,000,000
North11,500,0001,500,000
South1650,000650,000
Same idea, different names
SELECT department, COUNT(*) AS headcount, AVG(salary) AS avg_salary FROM employees GROUP BY department;
Different data, same shape. Group by department instead of region. Count employees instead of leads.
Part 5
ORDER BY + LIMIT
Sort your results and control how many rows to show.
The pattern
SELECT columns FROM table ORDER BY column DESC LIMIT number;
What each word does
ORDER BY Sorts the result. Must come near the end of your query (after WHERE and GROUP BY).
DESC Largest first (descending). Think: Down.
ASC Smallest first (ascending). Think: Arrow up. This is the default — if you don't write anything, SQL assumes ASC.
LIMIT Only return this many rows. "Give me the top 3" = LIMIT 3.
-- text Two dashes start a comment. SQL ignores everything after -- on that line. It's just a note to yourself.
Clicking a column header in Excel to sort A→Z or Z→A, then deleting all rows below a certain point. ORDER BY sorts, LIMIT cuts.
Example
SELECT lead_name, deal_value FROM leads ORDER BY deal_value DESC LIMIT 3; -- top 3 deals only
#lead_namedeal_value
1Arjun Nair1,500,000
2Karan Bose1,200,000
3Riya Mehta800,000
Part 6
JOIN — Combining two tables
When your data lives in two separate tables, JOIN connects them using a column they share.

Imagine you have two Excel sheets. One has lead info, the other has campaign info. They're connected by a shared ID — like a code that appears in both sheets.

The blue IDs are the link between the two tables. campaign_id in leads matches id in campaigns. This is how SQL knows which rows belong together.
The pattern
SELECT columns_from_either_table FROM first_table AS shortname1 JOIN second_table AS shortname2 ON shortname1.shared_column = shortname2.shared_column;
What each word does
JOIN Connects two tables. Short for INNER JOIN — only includes rows where a match exists in both tables.
ON Tells SQL which columns to match. "Connect rows where this column equals that column."
AS l / AS c Short nicknames for the tables. So you can write l.lead_name instead of typing the full table name every time.
l.column The dot means "from this table, get this column". Like saying "from the leads table, get lead_name".
VLOOKUP in Excel. You take campaign_id from the leads sheet, look it up in the campaigns sheet, and bring over the campaign_name and channel columns.
Example
SELECT l.lead_name, c.campaign_name, c.channel, l.deal_value FROM leads AS l JOIN campaigns AS c ON l.campaign_id = c.id;
Result — both tables merged into one

Same idea, different names
SELECT o.order_id, c.customer_name, o.amount FROM orders AS o JOIN customers AS c ON o.customer_id = c.id;
Orders + Customers instead of Leads + Campaigns. The pattern is the same.
Part 7
Putting it all together
A real-world query that uses JOIN + WHERE + GROUP BY + aggregate functions + ORDER BY — all at once.
Before we write the query, here's the order that SQL keywords must appear in:
SELECT
→
FROM
→
JOIN ... ON
→
WHERE
→
GROUP BY
→
ORDER BY
→
LIMIT
You don't always use all of them. But when you do, this is the order.
The question we want to answer:
"For deals over 7 lakhs, show me total leads and revenue by marketing channel — highest revenue first."
SELECT c.channel, COUNT(*) AS leads, SUM(l.deal_value) AS total_revenue FROM leads AS l JOIN campaigns AS c ON l.campaign_id = c.id WHERE l.deal_value > 700000 GROUP BY c.channel ORDER BY total_revenue DESC;
Reading it line by line
SELECT — I want to see the channel, a count of leads, and the sum of deal values.
FROM + JOIN — Get data from the leads table, connected to the campaigns table.
WHERE — But only keep deals worth more than 700,000.
GROUP BY — Group the remaining rows by channel.
ORDER BY DESC — Sort so the highest revenue channel is on top.

One last reminder

Blue words = SQL commands. Always the same. Memorise these.
Green words = table and column names. These come from YOUR data. They will be different every time.
Amber words = values you're searching for. These are whatever you need them to be.
Pink words = placeholders in patterns. Replace these with real names from your database.
